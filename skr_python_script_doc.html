1. 概述

本 Python 腳本旨在查詢 Solana 區塊鏈上的 Seeker (SKR) 代幣資訊。它整合了錢包內的液動代幣餘額查詢與自定義質押合約 (Custom Staking Program) 的質押餘額查詢與計算。

主要功能：

  • 查詢指定錢包的 SPL Token 餘額 (針對 SKR Mint)。
  • 查詢自定義質押合約中的用戶質押數據。
  • 自動抓取全域質押狀態以計算當前匯率 (Exchange Rate)。
  • 合併計算總資產並輸出格式化報告。

2. 環境需求

  • Python 版本: Python 3.7+
  • 依賴套件:
    • solana: 用於與 Solana RPC 互動。
    • solders: 提供高效能的 Solana 原語 (Pubkey, Keypair 等)。

安裝指令:

pip install solana solders

4. 函式功能詳解

4.1 get_token_accounts_by_owner

功能: 查詢標準 SPL 代幣餘額。

  • 輸入: wallet_address (str)
  • 邏輯:
    • 使用 get_token_accounts_by_owner RPC 方法。
    • 指定 TokenAccountOpts(mint=...) 進行過濾，僅回傳 SKR 代幣帳戶。
    • 使用 base64 編碼讀取以確保資料完整性。

4.2 get_staked_balance (核心邏輯)

功能: 計算用戶在質押合約中的實際 SKR 餘額。

  • 輸入: wallet_address (str)
  • 回傳: float (SKR 數量)
  • 演算法流程:
    1. 尋找用戶質押帳戶 (User Stake Account):
      • 使用 get_program_accounts 掃描質押合約。
      • Filter: Memcmp(offset=41, bytes=wallet_address)。這表示在 Account Data 的第 41 bytes 處比對錢包地址。
      • Fallback: 若 offset 41 找不到，嘗試 offset 40。
    2. 讀取用戶份額 (User Shares):
      • 從找到的帳戶 Data 中讀取 Offset 104 ~ 112 (Uint64, Little Endian)。
    3. 讀取全域狀態 (Global State):
      • 查詢 GLOBAL_STATE_PUBKEY。
      • 讀取 Total Staked (Offset 3616) 與 Total Shares (Offset 1344)。
    4. 計算實際餘額:
      Balance=User Shares×Total StakedTotal Shares\text{Balance} = \frac{\text{User Shares} \times \text{Total Staked}}{\text{Total Shares}}Balance=Total SharesUser Shares×Total Staked​
    5. 精度轉換:
      • 質押合約精度為 9 位小數 (10^9)。
      • 注意：這與錢包餘額 (6 位小數) 不同。

4.3 check_skr_staking

功能: 主流程控制與輸出。

  • 呼叫上述兩個函式。
  • 解析 SPL Token 的 Raw Data (Uint64) 並除以 10610^6106 (SKR Decimals)。
  • 加總並列印詳細報告。

5. 異常處理

腳本包含多層 try-except 區塊以處理常見錯誤：

  • RPC 連線錯誤: 當公共節點逾時或拒絕連線時，回傳 0 或 None 並印出錯誤訊息。
  • 地址格式錯誤: 使用 Pubkey.from_string 驗證輸入參數。
  • 解析錯誤: 處理 Base64 解碼與 Index Error (當回傳資料長度不足時)。

  • Total Staked Offset: 目前為 3616
  • Total Shares Offset: 目前為 1344
