Seeker Staking Checker

Platform Architecture & Mechanism Guide
Filename: skr_staking_checker.py Version: 1.0.0 Updated: 2026-02-02

1. Overview

This Python script is designed to query Seeker (SKR) token information on the Solana blockchain. It integrates liquid token balance queries within a wallet and calculates staking balances from the Custom Staking Program.

Key Features:

  • Query specific wallet's SPL Token balance (targeted for SKR Mint).
  • Query user staking data from the custom staking contract.
  • Automatically fetch global staking state to calculate the current Exchange Rate.
  • Merge calculations for total assets and output a formatted report.

2. Requirements

  • Python Version: Python 3.7+
  • Dependencies:
    • solana: For interacting with Solana RPC.
    • solders: Provides high-performance Solana primitives (Pubkey, Keypair, etc.).

Installation Command:

pip install solana solders

4. Function Details

4.1 get_token_accounts_by_owner

Function: Query standard SPL token balance.

  • Input: wallet_address (str)
  • Logic:
    • Uses get_token_accounts_by_owner RPC method.
    • Specifies TokenAccountOpts(mint=...) to filter and return only SKR token accounts.
    • Uses base64 encoding to ensure data integrity.

4.2 get_staked_balance (Core Logic)

Function: Calculates the user's actual SKR balance in the staking contract.

  • Input: wallet_address (str)
  • Return: float (SKR Amount)
  • Algorithm Flow:
    1. Find User Stake Account:
      • Uses get_program_accounts to scan the staking contract.
      • Filter: Memcmp(offset=41, bytes=wallet_address). This matches the wallet address at the 41st byte of Account Data.
      • Fallback: If offset 41 is not found, try offset 40.
    2. Read User Shares:
      • Read Offset 104 ~ 112 (Uint64, Little Endian) from the found account Data.
    3. Read Global State:
      • Query GLOBAL_STATE_PUBKEY.
      • Read Total Staked (Offset 3616) and Total Shares (Offset 1344).
    4. Calculate Real Balance:
      Balance = (User Shares Ã— Total Staked) / Total Shares
    5. Precision Conversion:
      • Staking contract precision is 9 decimal places (10^9).
      • Note: This differs from the wallet balance (6 decimal places).

4.3 check_skr_staking

Function: Main process control and output.

  • Calls the above two functions.
  • Parses SPL Token Raw Data (Uint64) and divides by 10^6 (SKR Decimals).
  • Sums up and prints a detailed report.

5. Exception Handling

The script includes multi-layer try-except blocks to handle common errors:

  • RPC Connection Error: Returns 0 or None and prints an error message when the public node times out or refuses connection.
  • Address Format Error: Uses Pubkey.from_string to validate input parameters.
  • Parsing Error: Handles Base64 decoding and Index Errors (when return data length is insufficient).

  • Total Staked Offset: Currently 3616
